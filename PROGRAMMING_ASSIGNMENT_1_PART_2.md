# Programming Assignment 1, Part 2

**Weight:** 80%  
**Due date:** Thursday, April 9th

## Project Requirements

In this stage of the project, each group must implement the parser for their files.

Each project should include:

- Lexer and parser properly implemented in PLY.
- After parsing is done, information must be kept in memory using objects, data structures, or any other mechanism.
- The resulting application must allow for at least one specific functionality using the parsed data.

## Submission Requirements

Each team must submit:

- At least three complete sample input files.
- The CFG in a separate file, not in the code, and properly explained.
- At least 10 states of the LALR(1) canonical table of the CFG.
- A video showing how the parser works, including a demo and a code walkthrough.

## How To Submit

Submit a single PDF file with:

- A link to your code repository. Share the repo with your professor.
- The CFG.
- The partial canonical table.
- A link to the video. You may upload it wherever you prefer.
